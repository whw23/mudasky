## LSP 使用规范

优先使用 LSP 工具理解代码上下文和确认影响范围，而非仅靠 Grep/Read。简单场景（如只改注释、改字符串常量）可跳过。

### 修改代码前 — 理解上下文

| 场景                     | 推荐操作                                    |
|--------------------------|---------------------------------------------|
| 不熟悉某个函数/类的作用  | `hover` 查看类型和文档                      |
| 需要了解函数定义         | `goToDefinition` 跳转源码                   |
| 需要了解文件/模块结构    | `documentSymbol` 列出符号                   |
| 需要搜索某个符号         | `workspaceSymbol` 全局搜索                  |
| 需要理解调用关系         | `incomingCalls` / `outgoingCalls` 查调用链  |
| 需要了解接口实现         | `goToImplementation` 查实现类               |

### 重构/改名等操作 — 确认影响范围

修改函数签名、重命名、删除等操作前，优先使用 `findReferences` 确认所有引用点，防止遗漏。
